# HIVE OS Complete Installation & Usage Guide\n\n## Quick Start\n\n### For Gentoo Linux (Full System)\n\n```bash\n# 1. Clone the repository\ngit clone -b feature/complete-packages https://github.com/JACKSCHITT-INC/THE-HIVE-OS.git\ncd THE-HIVE-OS\n\n# 2. Build core components\nmake clean && make\nmake test  # Run hive-test\n\n# 3. Set up Portage overlay\nsudo mkdir -p /var/db/repos/hive-os\nsudo cp -r overlay/* /var/db/repos/hive-os/\n\n# 4. Register overlay\nsudo cat > /etc/portage/repos.conf/hive-os.conf << 'EOF'\n[hive-os]\nlocation = /var/db/repos/hive-os\nauto-sync = no\nEOF\n\n# 5. Install HIVE meta-package (pulls all components)\nsudo emerge --sync hive-os\nsudo emerge hive-os/hive-meta\n\n# 6. Create hive system user/group\nsudo groupadd -r hive\nsudo useradd -r -g hive -d /var/lib/hive -s /sbin/nologin hive\n\n# 7. Initialize your HIVE node\nsudo hive-init --type GENTOO --sovereign \"YOURNAME\" --role \"System Architect\"\n\n# 8. Enable services\nsudo systemctl daemon-reload\nsudo systemctl preset 60-hive  # Auto-enable HIVE services\nsudo systemctl enable hive-personality hive-audit hive-collective-sync\nsudo systemctl start hive-personality hive-audit\n\n# 9. Login as your user and verify\nsu - YOURNAME\n# Should show: [HIVE:GENTOO:System Architect]$ prompt\n```\n\n### For Termux (Mobile Android)\n\n```bash\n# On Android device with Termux installed\npkg Foundation\n- `hive-init` - Bootstrap new HIVE nodes\n- `hive-shell` - Custom shell wrapper with personality context\n- `libhive.so` - Personality engine library\n- Configuration directories in `/etc/hive/`\n\n**hive-personality** - Evolution Engine\n- Continuous personality trait evolution daemon\n- Trait matrix configuration (8 personality dimensions)\n- Role hierarchy management (5 tiers)\n- Collective state synchronization\n\n**hive-kernel-module** - Kernel Enforcement\n- Linux Security Module (LSM) implementation\n- Kernel-level personality verification\n- Automatic threat response at kernel level\n- Collective events shared with kernel\n\n**hive-systemd** (or **hive-openrc**) - System Integration\n- Service management and startup\n- Timer-based periodic tasks\n- Sleep/wake state handlers\n- System-wide presets\n\n**hive-audit** - Intelligence Logging\n- Collective audit trail (90-day retention)\n- Threat detection and automatic response\n- Event classification and severity assessment\n- Permanent architect decision logs\n\n**hive-zorg** - The Shadow (Security Agent)\n- Active threat hunting (15-minute cycles)\n- Vulnerability scanning (6-hour cycles)\n- Zero-trust policy enforcement\n- Incident response automation\n\n**hive-pam** - Authentication Layer\n- PAM module for system-auth, SSH, su\n- Personality-aware authentication\n- Role-based access control\n- Authentication auditing\n\n---\n\n## Configuration\n\n### Personality Traits\n\nEdit `/etc/hive/traits/trait-matrix.conf`:\n\n```conf\n[TRAITS]\nauthority = 0.8        # Command exist\nsudo ls -la /etc/hive/\nsudo ls -la /var/lib/hive/\nsudo ls -la /var/log/hive/\n\n# Check file permissions\nsudo ls -la /etc/hive/personality/\n```\n\n### Shell Prompt Not Showing HIVE Identity\n\n```bash\n# Check if personality file exists for user\nsudo cat /etc/hive/personality/YOURNAME.conf\n\n# Re-initialize node\nsudo hive-init --type GENTOO --sovereign \"YOURNAME\" --role \"System Architect\"\n\n# Verify hive-shell can load it\nsudo -u YOURNAME hive-shell --version\n```\n\n### Collective Sync Not Working\n\n```bash\n# Check sync permissions\nsudo ls -la
